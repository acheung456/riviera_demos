Overview

What this endpoint does

Async workflow

/2/riviera/get_transcript_async starts a transcript job for media supplied as a Dropbox path, Dropbox file ID, or public URL. The endpoint can return a completed result immediately, or an async_job_id that you poll with /2/riviera/get_transcript_async/check.

This page is a product-style reference layer. Dropbox remains the source of truth for the official contract.
Authentication

Browser-friendly auth notes

The live demo in this repo uses Dropbox OAuth with PKCE. For static browser requests, we send the access token in the request URL using Dropbox's CORS-safe pattern rather than the usual Authorization header.

  • Scopes used by the demo: files.content.read and files.content.write
  • URL mode sends the remote media URL directly to Riviera
  • File mode uploads to Dropbox first and then uses the returned file identifier
Input union

file_id_or_url can take three forms

Tag
Shape
When to use it
path
{ ".tag": "path", "path": "/Apps/..." }
Useful when the media already lives in Dropbox.
file_id
{ ".tag": "file_id", "file_id": "id:..." }
Best after an upload step where Dropbox returns a stable file identifier.
url
{ ".tag": "url", "url": "https://..." }
Ideal for public media that Dropbox can fetch remotely.
Operation

POST Start transcript job

Primary endpoint

Send a request body with file_id_or_url and timestamp_level. In this demo, the selectable timestamp levels are word and sentence.

Required body
{
  "file_id_or_url": {
    ".tag": "url",
    "url": "https://archive.org/download/theravens_2603.poem_librivox/01%20-%20The%20Ravens%20-%20read%20by%20AH.mp3"
  },
  "timestamp_level": "sentence"
}
Operation

POST Check transcript job

Polling endpoint

If the launch call returns an async_job_id, poll the check endpoint with that identifier. The current demo waits at least fifteen seconds between checks and treats 429 as retryable.

Check body
{
  "async_job_id": "1234567890"
}
Responses

Common response shapes

Immediate success

.tag = "complete" with transcript content in the response.

Async launch

.tag = "async_job_id" plus async_job_id.

Still running

.tag = "in_progress" from the check endpoint.

Failure

.tag = "failed" or a non-2xx API error payload.

Errors and retries

What the demo handles for you

  • 429 Too Many Requests is treated as retryable on both Riviera endpoints
  • The check endpoint is spaced at a minimum fifteen-second interval
  • Raw JSON is surfaced in the UI to help debug failed requests
  • CORS-safe request formatting is used for the browser-based demo flow
