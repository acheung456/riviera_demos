Implementation notes

How this static demo works

OAuth setup

Register this page URL as a Dropbox redirect URI and provide the app key through the Pages workflow configuration. The deployed site uses PKCE, so no client secret is exposed in the browser.

URL mode

Sends { ".tag": "url", "url": "..." } in file_id_or_url and starts the async transcript job directly.

File mode

Uploads the file to Dropbox first, then uses the returned file_id to start transcription. That keeps the demo browser-only and GitHub Pages compatible.

Polling

Checks the async job until Riviera returns complete, failed, or an unexpected state.
